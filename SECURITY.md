# Security Policy

## Supported Versions

Security updates are currently provided only for the latest stable release.

| Version | Supported |
| ------- | --------- |
| Latest release | :white_check_mark: |
| Older releases | :x: |
| Development / forks | :x: |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it privately through one of these channels:

- **GitHub Security Advisories:** use the **"Report a vulnerability"** option in the repository’s **Security** tab.

If possible, please include:

- a clear description of the issue
- steps to reproduce it
- the expected impact
- the affected version
- operating system details
- screenshots, logs, or a minimal proof of concept (if applicable)

## What to Expect

I will do my best to:

- acknowledge receipt of the report within **5 business days**
- assess the validity and impact of the issue
- keep you informed during the process
- prepare and publish a fix as soon as possible

If the report is accepted as a valid vulnerability:

- I will work on a fix
- I will coordinate responsible disclosure
- I will credit you publicly, if you would like

If the report is not considered a security vulnerability, I will explain why whenever possible.

## Disclosure Policy

Please do not publicly disclose the vulnerability until a fix is available or a disclosure date has been agreed upon.

This helps protect users of the project.

## Scope

This policy covers vulnerabilities related to the security of the project, for example:

- unauthorized code execution
- authentication or authorization bypass
- exposure of credentials, tokens, or sensitive data
- command injection or unauthorized system access
- issues related to app packaging or updates

This policy does not cover:

- functional bugs without a security impact
- minor usability issues
- general support or configuration questions

## Notes

This project is maintained in the author’s available time, so response and resolution times may vary. However, security reports will be handled as quickly as possible.
